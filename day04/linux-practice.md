# ⚙️ Process Checks

## 1. Display Running Processes

```bash id="9m3r6o"
ps aux
```

### Purpose

Displays all currently running processes along with:

* CPU usage
* memory usage
* process owner
* PID

---

## 2. Real-Time Process Monitoring

```bash id="3f8j2x"
top
```

### Purpose

Provides real-time system monitoring including:

* active processes
* CPU usage
* memory usage
* system load

---

## 3. Search for a Specific Process

```bash id="y7q4ne"
pgrep systemd
```

### Purpose

Searches for process IDs matching the given process name.

---

# 🔧 Service Checks

For this practice session, the `ssh` service was inspected.

---

## 4. Check Service Status

```bash id="mbx8dp"
systemctl status ssh
```

### Purpose

Displays:

* service state
* uptime
* logs
* process information

### Observation

The SSH service was active and running successfully.

---

## 5. List Running Services

```bash id="g5z0rd"
systemctl list-units --type=service
```

### Purpose

Lists active systemd services currently running on the system.

---

## 6. Check if Service is Enabled at Boot

```bash id="rk5n3a"
systemctl is-enabled ssh
```

### Purpose

Checks whether the service automatically starts during system boot.

---

# 📜 Log Checks

## 7. View Logs for SSH Service

```bash id="l4xj9v"
journalctl -u ssh
```

### Purpose

Displays logs generated by the SSH service.

Useful for:

* debugging service issues
* checking authentication events
* monitoring failures

---

## 8. View Last 50 Log Entries

```bash id="k3v7pl"
tail -n 50 /var/log/syslog
```

### Purpose

Displays the most recent 50 log entries from the system log file.

# 🛠️ Mini Troubleshooting Flow

## Scenario

Verify whether the SSH service is running correctly.

### Step 1 — Check Running Process

```bash id="b2d8wa"
ps aux | grep ssh
```

Confirmed that the SSH process was running.

---

### Step 2 — Check Service Status

```bash id="d4t6py"
systemctl status ssh
```

Verified the service was active.

---

### Step 3 — Inspect Service Logs

```bash id="w7e1kn"
journalctl -u ssh
```

Reviewed recent SSH-related log entries for warnings or failures.
